# HypeAI 경쟁력 분석 & 전략 보고서

## 1. 시장 분석

### 1.1 경쟁 환경

| 서비스 | 특징 | 한계 |
|--------|------|------|
| GitHub Copilot Chat | IDE 내 AI 코딩 도우미 | IDE 필수, 모바일 불가 |
| Cursor | AI-native 코드 에디터 | 데스크톱 전용 |
| Cline (VS Code) | Claude 기반 자율 에이전트 | VS Code 필수 |
| Aider | 터미널 기반 AI 코딩 | CLI 전문 지식 필요 |
| Sweep.dev | GitHub PR 자동 생성 | 코드 리뷰 기능 부재 |

### 1.2 HypeAI 포지셔닝

> **"모바일에서 AI 개발을 관리하는 유일한 도구"**

기존 경쟁자들은 모두 데스크톱/IDE 환경을 전제로 한다.
HypeAI는 Discord를 인터페이스로 활용하여 **모바일에서도 AI 개발 작업을 지시하고 결과를 확인**할 수 있는 유일한 솔루션이다.

## 2. 핵심 차별점

### 2.1 모바일 퍼스트 코드 리뷰
- 대부분의 개발 도구는 데스크톱 전용
- Discord 모바일 앱을 통해 **언제 어디서나 코드 리뷰** 가능
- 모바일 최적화 Diff 렌더링 (480px, 14px 폰트)

### 2.2 멀티 AI 세션 관리
- Claude, Gemini, OpenCode를 동시에 운영
- 세션 간 전환으로 **다중 프로젝트/다중 AI** 관리
- 세션 영속성: 봇 재시작 후에도 세션 유지

### 2.3 보안 내장
- 민감 파일 자동 필터링 (.env, credentials 등)
- 출력 새니타이징 (API 키, 토큰 자동 마스킹)
- 작업 디렉터리 제한
- 명령어 차단 목록

### 2.4 태스크 큐
- 작업 목록 등록 후 순차 자동 실행
- 모바일에서 여러 작업을 예약하고 결과 확인

## 3. 전략적 로드맵

### Phase 1: 코드 리뷰 강화 (현재)
- 모바일 최적화 Diff 렌더링
- 파일별 분할 탐색
- `!review` 명령어 (AI 요약 + 인터랙티브 리뷰)
- 리뷰 액션 (승인/수정 요청/코멘트)

### Phase 2: 워크플로우 자동화
- PR 자동 생성 (`!pr` 명령어)
- 테스트 실행 및 결과 보고
- CI/CD 연동

### Phase 3: 협업 기능
- 팀 단위 세션 공유
- 리뷰 결과 요약 알림
- 작업 할당 및 추적

### Phase 4: 생태계 확장
- 플러그인 시스템
- 커스텀 AI 프로바이더 지원
- 웹 대시보드

## 4. 타겟 사용자

### 주요 타겟
1. **1인 개발자** — 이동 중 AI 작업 관리
2. **소규모 팀** — 모바일에서 빠른 코드 리뷰
3. **사이드 프로젝트 개발자** — 출퇴근 시간 활용

### 사용 시나리오
- 출근길에 Discord로 AI에게 작업 지시
- 점심시간에 결과 확인 및 코드 리뷰
- 귀가 중 수정 요청 및 추가 작업 지시
- 자기 전 최종 결과 확인 및 커밋 승인

## 5. 기술적 강점

| 영역 | 기술 | 이점 |
|------|------|------|
| 렌더링 | Puppeteer + diff2html | 고품질 Diff 이미지 |
| AI 연동 | Claude Agent SDK | 네이티브 세션 관리 |
| UI | Discord.js Buttons/Menus | 풍부한 인터랙션 |
| 보안 | 다계층 필터링 | 민감 정보 보호 |
| 배포 | exe 번들링 | 설치 없이 실행 |

## 6. 결론

HypeAI는 "모바일에서 AI 개발 관리"라는 미개척 시장을 타겟으로 한다.
Discord의 플랫폼 장점(모바일 앱, 봇 생태계, 인터랙티브 UI)을 활용하여
기존 IDE 기반 도구가 제공하지 못하는 **이동 중 개발 관리 경험**을 제공한다.

코드 리뷰 기능의 모바일 최적화는 이 비전의 첫 번째 구체적 실행이며,
"Summary First, Details on Demand" 원칙을 통해 모바일 환경의 제약을 극복한다.
